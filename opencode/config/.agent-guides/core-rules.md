# Core Rules

모든 작업에 공통 적용되는 글로벌 핵심 규칙입니다.

## MUST Rules

- **한국어 소통, 영문 코딩**: 대화/문서는 한국어를 기본으로 하고, 코드/주석은 영어를 기본으로 작성합니다.
- **안전 우선**: 파일 삭제, `git push -f` 같은 파괴적 작업은 사용자 명시 승인 후에만 수행합니다.
- **검증 필수**: 코드 작성 후 실행 또는 테스트로 동작을 검증합니다.
- **컨텍스트 유지**: 전후 맥락을 유지하고 불필요한 중복 질문을 피합니다.
- **TDD 준수**: 기능 구현 전에 테스트 코드를 먼저 작성합니다.
- **테스트 격리**: 테스트는 프로젝트와 격리된 시스템 임시 디렉토리에서 수행합니다.

## Communication Preference

- 응답은 명확하고 간결하게 작성하고, 필요 시 예시를 제공합니다.
