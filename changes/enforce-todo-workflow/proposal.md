# OpenSpec Proposal: 에이전트 워크플로우 내 Todo List 작성 의무화

## 1. 배경 및 목적 (Background & Objectives)
현재 에이전트들은 작업을 수행할 때 명시적인 계획 수립 단계 없이 바로 실행에 들어가는 경우가 있습니다. 이는 복잡한 작업에서 맥락을 잃거나 진행 상황을 파악하기 어렵게 만듭니다.
이 제안은 모든 에이전트가 작업 시작 시 `todowrite` 도구를 사용하여 Todo List를 작성하고, 작업 진행에 따라 이를 지속적으로 업데이트하도록 강제하여 작업의 투명성과 체계성을 높이는 것을 목표로 합니다.

## 2. 변경 범위 (Scope of Changes)
- **대상 파일**: `opencode/config/agent/*.md` (총 10개 파일)
- **변경 내용**:
    1.  `tools` 섹션에 `todowrite: true`, `todoread: true` 추가 (미존재 시).
    2.  `Workflow` 또는 `Core Principles` 섹션에 "Todo List 기반 작업 관리" 지침 추가.

## 3. 기대 효과 (Expected Benefits)
- 에이전트의 작업 계획 능력 향상
- 사용자가 에이전트의 현재 진행 상황과 남은 작업을 명확히 파악 가능
- 복잡한 작업 수행 시 에이전트의 맥락 유지 능력 강화

## 4. 리스크 (Risks)
- 단순한 작업(단일 질문 답변 등)에서도 Todo List 작성을 시도하여 오버헤드가 발생할 수 있음. (지침에 "복잡한 작업 시" 또는 "워크플로우 시작 시"라는 조건을 명시하여 완화 필요)
