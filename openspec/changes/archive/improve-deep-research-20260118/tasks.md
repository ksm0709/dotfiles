## 검증 방법

### 단위 테스트
- 각 클래스별 기능 테스트
- 엣지 케이스 및 예외 상황 테스트
- Mock 객체를 통한 격리 테스트

### 통합 테스트
- 전체 파이프라인 실행 테스트
- 다양한 환경 조건에서의 동작 검증
- API 키 없는 상황에서의 fallback 테스트

### 수동 검증
- 실제 API 키로 동작 확인
- 다양한 OS 환경에서 테스트
- 사용자 시나리오 기반 검증

## 의존성
- Python 3.8+ (기존과 동일)
- 기존 의존성 패키지 유지
- 새로운 외부 의존성 없음

- 리팩토링: 2시간
- 테스트: 2시간
- 문서화: 1시간
